# Vibe Coding Best Practice Guide

VibeCodingでアプリをローンチするためのベストプラクティスを、実際にアプリを作りながら学ぶための研修リポジトリです。

---

## このリポジトリについて

「数週間で終わるはずのものに数ヶ月かけてしまう」パターンを断ち切るための、**理論ではなく実際に効果があるツール・手法**を体験的に学びます。

サンプルアプリ **Folio**（デザイナー・イラストレーター向け作品公開・販売サイト）をPhaseごとに実装しながら、各ツールを「なぜ使うのか」を体感することが目的です。

## 学習の進め方

### 1. まず原則を読む
`01_principles/vibe-coding-guide-ja.md` を読み、何を使い・何を作らないかの考え方を把握する。

### 2. アプリ仕様を確認する
`02_app-spec/folio-spec.md` で全体像・技術スタック・データモデルを把握する。

### 3. Phaseを順番に進める
`03_curriculum/` のガイドに沿って、1Phaseずつ実装する。各Phaseが完了したら次のPhaseへ。

## サンプルアプリ：Folio

| 項目 | 内容 |
|---|---|
| 概要 | デザイナー・イラストレーターが作品を公開・販売する個人サイト |
| ユーザー | オーナー（自分）と訪問者（潜在顧客）の2種類 |
| 主な機能 | 作品のCRUD・画像アップロード・Stripe決済 |
| 月額費用 | $0（Stripeは売上が発生したときのみ手数料） |

---

## 使用ツール一覧

| ツール | 役割 | 無料枠 |
|---|---|---|
| Next.js | フレームワーク | 完全無料（OSS） |
| Tailwind CSS + shadcn/ui | UI | 完全無料（OSS） |
| Supabase | DB + 認証 | 永久無料枠あり |
| Prisma | ORM | 完全無料（OSS） |
| UploadThing | ファイルアップロード | 永久無料（2GB） |
| Stripe | 決済 | 手数料のみ |
| Sentry | エラー監視 | 永久無料枠あり |
| PostHog | 分析 | 永久無料枠あり |
| Vercel | デプロイ | 永久無料枠あり |
