## エージェント向け指示

このドキュメントの各Stepには実施者ラベルが付いています。

## Server Components と Client Components について

このPhaseで最も重要な概念。エージェントへの指示に影響するため、理解してから進めること。

|                      | Server Components              | Client Components           |
| -------------------- | ------------------------------ | --------------------------- |
| デフォルト           | ✅ App Routerのデフォルト      | `"use client"` を先頭に書く |
| DBアクセス           | ✅ できる                      | ❌ できない                 |
| useState / useEffect | ❌ 使えない                    | ✅ 使える                   |
| ブラウザイベント     | ❌ 使えない                    | ✅ 使える                   |
| 用途                 | ページ・データ取得・レイアウト | インタラクティブなUI部品    |

**原則：** できるだけServer Componentsを使い、インタラクションが必要な部分だけClient Componentsにする。

## Step 4 `[AGENT]`：空状態コンポーネントを作成する

### やること

作品が1件もない場合に表示する `EmptyState` コンポーネントを作成する。

### なぜ空状態が重要なのか

作品が0件のとき、何も表示しないと「バグ？それとも正常？」とユーザーが混乱する。適切なメッセージと次のアクション（「作品を追加する」ボタン）を示すことで、ユーザーを迷わせない。

### 要件

- ファイル：`components/empty-state.tsx`
- props：`title`・`description`・`action`（オプション：ボタンのラベルとhref）
- 訪問者向け：「まだ作品がありません」
- 管理者向け：「作品を追加してみましょう」＋「作品を追加」ボタン
- Server Componentで実装する

### エージェントへの指示例

> `components/empty-state.tsx` を作成してください。title・description・actionのpropsを受け取り、作品が0件のときに表示する空状態UIです。actionが渡された場合はボタンも表示します。

### 確認ポイント

- `components/empty-state.tsx` が存在すること

## よくある失敗パターン

**不要な場所で `"use client"` をつけてしまう**
表示だけのコンポーネントにはServer Componentsを使う。`"use client"` は `useState`・`useEffect`・ブラウザイベントが必要なときだけ使う。エージェントはデフォルトで `"use client"` をつけがちなので、指示で明示すること。

**生のCSSを書いてしまう**
shadcn/uiにないスタイルが必要な場合もTailwindのユーティリティクラスで対応すること。`style={{}}` や `.css` ファイルは使わない。

**Phase 4の作業を先にやってしまう**
このPhaseではDBには触らない。「ダミーデータで動く状態」が完成してからPhase 4に進むこと。
